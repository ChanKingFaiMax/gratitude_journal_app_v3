# 感恩日记应用设计文档

## 应用概述
一个帮助用户养成感恩习惯的移动应用,通过每日提示、写作辅助和完成报告来引导用户记录感恩日记。

## 屏幕列表

### 1. 首页 (Home)
**主要内容:**
- 今日日期显示
- 今日推荐题目卡片(3-5个可选题目)
- 快速开始写作按钮
- 已完成日记的历史列表(最近7天)

**功能:**
- 点击题目卡片开始写作
- 查看历史日记条目
- 刷新获取新题目

### 2. 写作页面 (Write)
**主要内容:**
- 顶部显示选中的题目
- 大文本输入区域
- 底部工具栏:
  - "需要灵感"按钮(获取写作提示)
  - 字数统计
  - 完成按钮

**功能:**
- 自由文本输入
- 点击"需要灵感"显示写作提示(根据当前题目生成)
- 完成后跳转到报告页面

### 3. 报告页面 (Report)
**主要内容:**
- 祝贺信息
- 本次写作统计(字数、用时)
- AI生成的简短总结(2-3句话,提炼感恩主题)
- 情感分析(积极度指标)
- 返回首页按钮

**功能:**
- 展示本次日记的完成情况
- 提供正向反馈

### 4. 历史页面 (History) - Tab
**主要内容:**
- 日历视图,标记已完成日期
- 按日期倒序的日记列表
- 每条显示:日期、题目、摘要

**功能:**
- 点击查看完整日记内容
- 筛选和搜索

### 5. 设置页面 (Settings) - Tab
**主要内容:**
- 每日提醒时间设置
- 主题切换(浅色/深色)
- 关于应用

**功能:**
- 配置通知时间
- 切换外观

## 关键用户流程

### 流程1: 开始新日记
1. 用户打开应用 → 首页
2. 浏览今日推荐题目
3. 点击感兴趣的题目 → 写作页面
4. 开始输入文字
5. (可选)点击"需要灵感"获取写作提示
6. 完成写作,点击"完成"按钮 → 报告页面
7. 查看报告后返回首页

### 流程2: 查看历史日记
1. 用户打开应用 → 首页
2. 点击底部Tab切换到"历史"
3. 浏览日历或列表
4. 点击某条日记查看详情

### 流程3: 设置提醒
1. 用户打开应用 → 首页
2. 点击底部Tab切换到"设置"
3. 点击"每日提醒时间"
4. 选择时间 → 保存

## 色彩方案

**主色调:**
- Primary: 温暖的橙色 `#FF8C42` (象征感恩和温暖)
- Secondary: 柔和的黄色 `#FFD166` (积极向上)

**背景色:**
- Light: `#FFFBF5` (米白色,温馨)
- Dark: `#1A1A1A` (深灰色)

**文字色:**
- Light模式: `#2D2D2D` (深灰)
- Dark模式: `#F5F5F5` (浅灰)

**辅助色:**
- Success: `#06D6A0` (完成状态)
- Muted: `#8B8B8B` (次要信息)

## 设计原则

1. **简洁直观**: 减少认知负担,让用户专注于写作
2. **温暖友好**: 使用圆角、柔和色彩营造舒适氛围
3. **单手操作**: 主要操作按钮位于屏幕下半部分
4. **即时反馈**: 每个操作都有明确的视觉反馈
5. **无压力**: 不强制每日完成,鼓励而非强迫

## 技术实现

- 本地存储: AsyncStorage (日记内容、用户设置)
- AI功能: 使用内置LLM生成写作提示和报告总结
- 通知: expo-notifications (每日提醒)
- 动画: react-native-reanimated (页面转场、按钮反馈)
